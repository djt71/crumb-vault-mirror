### Cache Investigation

**Why cache didn't activate:**
Haiku 4.5 minimum cacheable prompt size = **4,096 tokens**. Our compressed voice prompt
(~1,090 tokens) is below the threshold. API processes requests without caching; no error.

In production through OpenClaw, system prompt = ~6,590 tokens (voice prompt + ~4,000 tool
definitions + ~1,500 system instructions) → exceeds 4,096 minimum. Caching activates.

### Revised Cost Model

**Critical insight:** Spec's original model assumed 135 requests/day to Anthropic including
96 heartbeats. In production two-agent architecture, heartbeats go to Ollama (free). Only
~39 manual Telegram interactions go to Anthropic.

**Cache economics:** Write premium ($1.25/M) vs uncached ($1.00/M) means caching costs MORE
on misses. Break-even: ≥22% hit rate. With default 5-min TTL and ~3.9 requests/hour: ~50%.

**Monthly projections (39 voice requests/day):**

| Scenario | Cache hit rate | Monthly cost | vs $8.70 target |
|----------|---------------|-------------|----------------|
| Uncached | — | $10.77 | +24% |
| Short TTL (5 min) | ~50% | $8.40 | -3% |
| Long TTL (1 hour) | ~75% | $6.09 | -30% |

**Verdict:** Cost model confirmed within ±20% at moderate cache hit rates. $8.40/mo with
default OpenClaw caching. Original model was wrong about mechanism (all traffic on API +
100% cache reads) but right about the number — two-agent split compensates for realistic
cache rates.
