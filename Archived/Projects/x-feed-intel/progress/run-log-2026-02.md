- **Pattern: stem-based path family.** A single stem (`research-A01-karpathy-slug-20260225`) drives investigate file, context file, and research output paths via suffix/directory variation. Keeps related artifacts visually grouped in `ls` output and simplifies path derivation in completion/recovery code. Convention, not a reusable primitive.

- **macOS launchd + provenance xattr:** `com.apple.provenance` xattr on copied files causes `launchctl bootstrap` to fail with "Input/output error" on macOS 15+. Strip with `xattr -d` before bootstrap. This is a macOS platform gotcha worth documenting alongside the existing openrsync and `set -e` notes.
- **`launchctl list` blind spot:** Calendar-interval jobs that exit cleanly (exit code 0) don't appear in `launchctl list`. Use `launchctl print gui/$(id -u)/<label>` instead. Misleading for operators who rely on `list` as the health check.
- **Credential consolidation pattern:** When multiple services share the same credential, store it once (Keychain) and have all consumers read from there. Eliminates rotation as a multi-step procedure.
