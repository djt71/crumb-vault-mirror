# Code Review: M4 — YouTube Adapter & Heavy-Tier Triage

## Anthropic — Claude Opus 4.6 (ANT)

### Findings

**ANT-F1** | SIGNIFICANT
- **File:** `src/adapters/yt/normalizer.ts:64-73`
- **Finding:** `parseIsoDuration` silently returns `null` for unrecognized formats without logging.
- **Why:** If YouTube returns an unexpected format, videos silently lose duration data. Makes debugging content tier decisions harder.
- **Fix:** Add `console.warn` when input is non-empty but regex doesn't match.

**ANT-F2** | CRITICAL
- **File:** `src/adapters/yt/index.ts:149-157`
- **Finding:** Circuit breaker state is per-pull-path (curated vs discovery), not shared. Two independent circuit breakers mean the adapter needs 3 failures on each path before disabling, doubling blast radius.
- **Why:** Alternating runs could never trip either breaker while consistently failing.
- **Fix:** Store circuit breaker state at the adapter level (shared), or in `adapter_state` table with component `'circuit_breaker'`.

**ANT-F3** | SIGNIFICANT
- **File:** `src/cli/deps/summarize-llm.ts:154-157`
- **Finding:** API key resolved once at factory creation time. If missing, entire process fails even when summarization isn't needed for that run.
- **Fix:** Document as intentional (fail-fast) or make lazy.

**ANT-F4** | SIGNIFICANT
- **File:** `src/cli/deps/yt-api.ts:30-31`
- **Finding:** API key appended to URL without URL-encoding; key could appear in error messages.
- **Fix:** Use `encodeURIComponent(apiKey)` and sanitize error output with `[REDACTED]`.

**ANT-F5** | SIGNIFICANT
- **File:** `src/adapters/yt/index.ts:162-232`
- **Finding:** `pullCurated` and `pullDiscovery` have nearly identical loop bodies (fetch details, dedup, fetch transcripts, normalize, update circuit breaker).
- **Why:** Bug fixes must be applied in two places — #1 source of future divergence.
- **Fix:** Extract shared `processVideoIds()` helper.

**ANT-F6** | SIGNIFICANT
- **File:** `src/cli/deps/summarize-llm.ts:107-131`
- **Finding:** `truncateWithHeadings` drops headings at truncation boundary rather than preserving them. Logic inverted from stated purpose.
- **Fix:** Include heading if it fits within small overflow tolerance.

**ANT-F7** | MINOR — `crypto` wildcard import for single `sha256` call.
**ANT-F8** | MINOR — `full_text_token_limit: 8000` uses 4 chars/token heuristic; worth a comment.
**ANT-F9** | MINOR — `jitterDelay()` on every transcript fetch including first; 50-100s pure delay per run.
**ANT-F10** | MINOR — Dedup test fixture setup is confusing; doesn't test cross-playlist dedup clearly.
**ANT-F11** | MINOR — Integration test couples to internal pipeline shape (fragile).
**ANT-F12** | MINOR — `fetchTranscript` always returns `isAutoGenerated: true` but preamble differentiates manual vs auto.
**ANT-F17** | MINOR — `(state as YtCuratedState) ?? {}` — `??` only triggers on null/undefined, technically unsound for non-object values.
**ANT-F18** | MINOR — Mock `SummarizeFn` intersection type repeated across test files.

**Strengths:**
- ANT-F13: Circuit breaker design (hard cap, error rate, consecutive-run threshold, auto-recovery).
- ANT-F14: `determineEffectiveTier` cleanly handles heavy-to-lightweight fallback.
- ANT-F15: YouTube preamble quality (ASR caveats, engagement weighting).
- ANT-F16: Full pipeline integration test with cost subcomponent assertions.

### Findings

**CDX-F1** | SIGNIFICANT
- **File:** `src/cli/deps/yt-api.ts:169-185`, `src/adapters/yt/index.ts:116-121`
- **Finding:** Circuit breaker error-rate tracking bypassed with real deps. `fetchTranscriptSafe()` increments errors only on thrown exceptions, but `createRealYtApiDeps().fetchTranscript()` catches all errors and returns `{available:false}`. In production, repeated failures won't trip breaker.
- **Fix:** Distinguish "no transcript" vs "operational failure" (e.g., `status: 'ok' | 'unavailable' | 'error'`) or let operational failures throw.

**CDX-F2** | SIGNIFICANT
- **File:** `src/cli/deps/yt-api.ts:138-151`, `src/adapters/yt/index.ts:174-177`
- **Finding:** Playlist pagination not implemented. `getPlaylistItems()` returns `nextPageToken` but adapter only fetches one page. Large playlists silently miss videos after first 50.
- **Fix:** Add `pageToken` support and loop pages (bounded by max budget).

**CDX-F3** | SIGNIFICANT
- **File:** `src/cli/deps/yt-api.ts:157-166`
- **Finding:** `getVideoDetails()` does not chunk video IDs to YouTube API max of 50.
- **Fix:** Split into chunks of 50, fetch each, merge results.

**CDX-F4** | MINOR — Unsafe casts (`as any` / unchecked `as YtTopicConfig`) bypass runtime validation.
**CDX-F5** | MINOR — `parseInt()` can yield `NaN` for malformed counts; downstream expects `number | null`.
**CDX-F6** | MINOR — `truncateWithHeadings()` behavior conflicts with its comment.
**CDX-F7** | MINOR — Circuit breaker tests only cover throw-based errors, masking F1 in production path.
**CDX-F8** | MINOR — Missing tests for playlist pagination and video ID chunking.

**Strengths:**
- CDX-S1: Effective-tier fallback (lightweight when transcript missing) aligns with design.
- CDX-S2: Good coverage of summarize integration, skip-on-`summary_generated`, cost subcomponent split.
