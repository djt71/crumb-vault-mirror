## Dependency Graph Summary

```
M1 (Framework Core) ──→ M2 (X Migration) ──→ M3 (RSS) ──→ M4 (YouTube)
                                   │                              │
                                   └──→ M5 (Remaining Adapters) ←─┘
                                         (HN/arxiv can start after M2;
                                          Reddit needs FIF-040 Phase 0)
```

Note: FIF-030 (RSS Phase 0) can start after FIF-005 (manifest loader) — does
not require M2 completion. RSS integration test (FIF-032) remains gated on M2
feature parity gate (FIF-029).

Note: M5 tasks depend on FIF-029 (feature parity gate), which transitively
requires all M1 infrastructure complete via the M1 → M2 chain.

Within M1, work packages can partially overlap:
- WP-1 (data layer) enables WP-2, WP-3, WP-4
- WP-2 (adapter mgmt) and WP-3 (triage) can proceed in parallel after WP-1
- WP-4 (output) depends on WP-3 triage output format
- WP-5 (observability) depends on WP-2 capture clock
- WP-6 (research promotion) depends on WP-4 feedback protocol

Within M2, staging environment setup (FIF-020) and X adapter refactoring (FIF-021)
can proceed in parallel after FIF-019. Migration script development (FIF-022–FIF-026)
requires staging environment. Live execution (FIF-028) requires scripts, rollback,
and integration test all complete.
